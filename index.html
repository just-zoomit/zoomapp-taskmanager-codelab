Supabase

Supabase is an open-source backend-as-a-service platform that offers features such as authentication, real-time databases, and RESTful APIs. It serves as a scalable alternative to Firebase and simplifies backend integration for web applications.

Ngrok

Ngrok is a tunneling tool used to expose local servers to the internet via secure HTTPS endpoints. In Zoom app development, Ngrok enables local testing by providing a publicly accessible URL‚Äînecessary because Zoom requires all app endpoints to use HTTPS.

In this section, you'll set up a Supabase project and associate a NextJs web app with it. You'll also set up the Supabase services used by the sample web app.

Create a Supabase project

1. Sign up

  • Visit supabase.com and sign up with your email or GitHub account. Once you have an account, log in to access your dashboard.

2. Create a new Project

  • In the Supabase dashboard, click on the New Project button. This initiates the project creation process.

3. Configure project settings

  • Project Name: Enter a name for your project.
  • Database Password: Set a strong password for the Postgres database that Supabase provisions.
  • Region: Choose the server region closest to your user base for optimal performance.

4. Provisioning the Database

After submitting the details, Supabase will create a new Postgres database instance. This process might take a couple of minutes. You'll see a progress indicator in your dashboard.

5. Access project details

Once the project is set up, navigate to the project's settings where you can find:

  • API Keys: Essential for authenticating requests from your client applications.
  • Connection Strings: Use these details to connect your application to the database.
  • Project URL: The base URL for accessing your project's API.

Set up your authentication

1. Find your Supabase callback URL and set up authentication

The next step requires a callback URL, which looks like this: https://.supabase.co/auth/v1/callback

  • Go to your Supabase Project Dashboard
  • Click on the Authentication icon in the left sidebar
  • Click on Providers under the Configuration section
  • Click on Zoom from the accordion list to expand and you'll find your Callback URL, you can click Copy to copy it to the clipboard

In this section, you'll install Supabase CLI for local development and connect your Supabase account.

Install Supabase CLI

  1. If you see a lower version or you don't have the Supabase CLI installed, run the install command:
brew install supabase/tap/supabase

Running Supabase locally

  1. Inside the folder where you want to create your project, run:
supabase init

This will create a new supabase folder. It's safe to commit this folder to your version control system.

  1. Now, to start the Supabase stack, run:
supabase start

This takes time on your first run because the CLI needs to download the Docker images to your local machine. The CLI includes the entire Supabase toolset, and a few additional images that are useful for local development (like a local SMTP server and a database diff tool).

Log in to Supabase

  1. Log in to the Supabase CLI and connect your Supabase account using your personal access token:
supabase login

Follow these steps to properly seed your Supabase database for development:

  1. Clear existing data:
    • Navigate to Supabase SQL Editor to clear migration history
    • Look for Clear Migration History Note: Remove local migration files and deletes the .snaplet folder for a complete reset
  2. Pull the latest database schema:supabase db pull
  3. Initialize Snaplet:npx @snaplet/seed init
  4. Sync Snaplet seed:npx @snaplet/seed sync
  5. Generate seed data:npx tsx seed.ts > supabase/seed.sql
  6. Verify generated data (optional):npx tsx seed.ts | grep sub_tasksnpx tsx seed.ts | grep tasks
  7. Reset and apply seeds:supabase db reset

Note: Make sure you have the necessary permissions and have backed up any important data before running these commands. db reset

In this section, you'll set up your Ngrok account and create an HTTPS tunnel for local development with Zoom Apps.

1. Configure Your ngrok Account:

  1. Sign up for a free ngrok account
  2. Obtain your authtoken from the ngrok dashboard
  3. Run the following command in your terminal, replacing YOUR_AUTH_TOKEN with your actual token:
    ngrok config add-authtoken YOUR_AUTH_TOKEN.

2. Start the ngrok Tunnel:

Zoom Apps do not support localhost, and must be served over https. To develop locally, you need to tunnel traffic to this application via https, because the application runs in Docker containers serving traffic from http://localhost. You can use Ngrok to do this.

  1. Once installed you may run this command from your terminal:
    ngrok http 3000

Ngrok will output the origin it has created for your tunnel, eg https://9a20-38-99-100-7.ngrok.io. You'll need to use this across your Zoom App configuration in the Zoom Marketplace (web) build flow (see below).

  1. Please copy the https origin from the Ngrok terminal output and paste it in the PUBLIC_URL value in the .env file.

Please note that this ngrok URL will change once you restart ngrok (unless you purchased your own ngrok pro account). If you shut down your Ngrok (there's no harm to leaving it on), upon restart you'll need to copy and paste the new origin into the .env file AND also to your Marketplace build flow.

3. Multiple Endpoints

  1. The ngrok configuration file allows you to define multiple https tunnels. To start multiple tunnels defined in the configuration file, use the following command:
ngrok start foo bar baz

Note: When a user is redirected via a deep link or manually opens your app from the Zoom client, the Zoom client sends a GET request to your app's Home URL, as configured in the Information tab of your Zoom App. Upon receiving a successful response, the Zoom client renders the returned content within the app interface.

This request includes the X-Zoom-App-Context header, which contains an AES-GCM‚Äìencrypted JSON payload. This payload includes details about the user and the context in which the app was launched.

Your app must decrypt the X-Zoom-App-Context header to extract this contextual information. Once decrypted, the payload reveals user identifiers, meeting identifiers, and other relevant data about the app session. Learn more about Zoom contextual data below.

Server and client components

The app is a Next.js web application that uses the App Router. Server rendering is applied throughout. For example, the src/app/page.js file is a server component that renders the main page. The src/components/tasks.jsx file is a client component, indicated by the "use client" directive at the top of the file.

Mock data

This project uses Snaplet to seed the database with realistic mock data generated using large language models. If your database schema changes, you must regenerate @snaplet/seed to synchronize with the updated structure. Run the following command to regenerate:

npx @snaplet/seed sync
npx tsx seed.ts > supabase/seed.sql
